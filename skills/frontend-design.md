# Frontend Design
Version: 1.0

Create distinctive, production-grade interfaces. Avoid generic AI aesthetics.

## Before Writing Code

1. **Identify purpose** — what problem does this UI solve? Who uses it?
2. **Choose a tone** — commit to one: brutally minimal / maximalist / retro-futuristic / editorial / brutalist / luxury / playful. Execute with precision.
3. **Identify the one memorable thing** — what will the user remember?

## Implementation Rules

- Working code only. No placeholders, no "TODO" comments in delivered UI.
- Responsive by default. Mobile-first unless specified otherwise.
- Accessible: semantic HTML, ARIA labels on interactive elements, keyboard navigable.
- Performance: no unnecessary dependencies. Vanilla JS where React is overkill.

## Quality Bar

Before delivering:
- [ ] Does it work? Test every interactive element mentally.
- [ ] Is the aesthetic intentional? Could describe the design direction in one word?
- [ ] Is it accessible? Screen reader navigable?
- [ ] Is it responsive? Works at 375px and 1440px?
- [ ] No console errors?
