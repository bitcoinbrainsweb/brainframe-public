## TOOL ROLES

### Claude
**Primary role:** Planning, architecture, critique, pressure-testing, document generation, Cursor prompt design.

**Use for:**
- Architecture decisions and critique
- Sequencing and roadmap review
- Generating Cursor prompts
- Generating Brainframe files and project documents
- Post-build critique
- Challenging assumptions

**Never use for:**
- Code implementation
- Direct interaction with Cursor or Base44
- Running terminal commands
- Autonomous commits or deploys

**Handoff format:** Load brainfile + master.md + relevant spec file. Claude receives context via paste.

---

### ChatGPT
**Primary role:** Long research tasks, operationalization, secondary opinion, Brainframe file generation when explicitly assigned.

**Use for:**
- Long compute research tasks while Dave is away
- Generating Brainframe system files when explicitly assigned by Dave
- Operationalizing plans into build-ready artifacts
- Secondary opinion on specific decisions
- Prompt pack generation when assigned

**Never use for:**
- Primary architecture decisions
- Overriding Claude's planning outputs without review
- Autonomous commits or deploys

**Handoff format:** Load brainfile + master.md + task-specific context. ChatGPT receives context via paste.

---

### Cursor
**Primary role:** All code implementation.

**Use for:**
- Writing and modifying code
- Repository-level changes
- Debugging
- Generating SANITYCHECK / REVIEW / RESULT artifacts

**Never use for:**
- Architecture decisions
- Planning or roadmap work
- Autonomous merges or deploys without Dave approval

**Prompt requirements:**
- Always include branch name
- Always include scope (what is in and out of scope)
- Always include expected artifacts
- Always define what success looks like
- Risky changes require REVIEW artifact before implementation

**Handoff format:** Cursor prompt generated by Claude, reviewed by Dave, pasted into Cursor.

---

### Make
**Primary role:** Brainframe file automation and staging commits.

**May do:**
- Generate files
- Stage git changes
- Prepare system artifacts
- Commit Brainframe file updates to staging branch only

**May never do:**
- Merge branches
- Deploy code
- Modify production infrastructure
- Commit to main without Dave approval

---

### Base44
**Primary role:** Rapid app prototyping.

**Use for:**
- Quick demos and proof of concept builds
- Projects where speed matters more than architectural purity
- Prototypes that will be rebuilt properly later

**Never use for:**
- Production systems unless explicitly decided per project
- Systems that require complex graph relationships or pipeline architecture at scale

---

## WORKFLOW SEQUENCE

Standard development flow:
```
Claude designs → Dave reviews → Cursor implements → Dave commits → Dave pushes → Dave deploys
```

Brainframe file update flow:
```
AI proposes update → Make stages to branch → Dave reviews → Dave merges to main → GitHub Actions syncs to project repos
```
